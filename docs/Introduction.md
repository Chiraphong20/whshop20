# บทนำและวัตถุประสงค์ (Introduction and Objectives)

เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายความเป็นมา วัตถุประสงค์ และขอบเขตการทำงานของระบบสั่งซื้อสินค้าออนไลน์สำหรับร้าน **WongHiran Shop (วงษ์หิรัญค้าส่ง 20 บาท)** ซึ่งเป็นการนำเทคโนโลยีมาประยุกต์ใช้เพื่อเพิ่มประสิทธิภาพในการบริหารจัดการร้านค้าและยกระดับประสบการณ์การสั่งซื้อของลูกค้าผ่านช่องทาง LINE

---

## 1. ความเป็นมา (Background)

ร้าน **WongHiran Shop (วงษ์หิรัญค้าส่ง 20 บาท)** เป็นธุรกิจค้าส่งและค้าปลีกสินค้าเบ็ดเตล็ดในราคาประหยัด ที่ผ่านมาทางร้านมีการรับออเดอร์จากลูกค้าผ่านหลายช่องทาง รวมถึงการสั่งซื้อผ่านแอปพลิเคชัน LINE ซึ่งเป็นช่องทางที่ลูกค้าคนไทยนิยมเข้าถึงมากที่สุด อย่างไรก็ตาม การรับออเดอร์ผ่านการแชทแบบเดิมมักพบปัญหาหลายประการ ได้แก่

- **ความล่าช้าในการตอบกลับและสรุปยอด:** แอดมินต้องใช้เวลาในการตรวจสอบสต็อกและสรุปยอดเงินให้ลูกค้าทีละราย
- **ข้อมูลออเดอร์ตกหล่นหรือผิดพลาด:** การจดบันทึกออเดอร์ด้วยมือมีโอกาสเกิดความผิดพลาดได้ง่าย
- **การจัดการสต็อกสินค้าที่ไม่เป็นระบบ:** ไม่มีระบบกลางเพื่อตัดสต็อกแบบเรียลไทม์ ทำให้บางครั้งลูกค้ารับทราบว่ามีสินค้า แต่เมื่อถึงเวลาจัดเตรียมสินค้าจริงกลับพบว่าสินค้าหมด
- **ความไม่สะดวกของลูกค้า:** ลูกค้าไม่สามารถดูรายการสินค้าทั้งหมดและราคารวมได้ด้วยตนเอง

ด้วยปัญหาดังกล่าว ทางร้านจึงมีแนวคิดในการพัฒนาระบบ **LINE Commerce** ซึ่งเป็นระบบ Web Application ที่ทำงานเชื่อมต่อกับแอปพลิเคชัน LINE เพื่อให้ลูกค้าสามารถเปิดดูแคตตาล็อกสินค้า เลือกสินค้าลงตะกร้า และยืนยันคำสั่งซื้อได้ด้วยตนเอง ในขณะที่แอดมินก็มีระบบหลังบ้าน (Admin Panel) เพื่อจัดการข้อมูลสินค้า ตรวจสอบออเดอร์ และรับการแจ้งเตือนเมื่อมีคำสั่งซื้อใหม่เข้ามาอย่างเป็นระบบ

---

## 2. วัตถุประสงค์ (Objectives)

การพัฒนาระบบ WongHiran Shop (LINE Commerce) มีวัตถุประสงค์หลักดังต่อไปนี้:

1. **เพื่อเพิ่มช่องทางการขายและอำนวยความสะดวกแก่ลูกค้า:** พัฒนาระบบสั่งซื้อสินค้าที่ใช้งานง่าย ลูกค้าสามารถดูรายการสินค้า เช็คราคารวม และกดสั่งซื้อได้ด้วยตนเองผ่าน LINE โดยไม่ต้องรอแอดมินตอบแชท
2. **เพื่อลดข้อผิดพลาดในการรับออเดอร์:** ระบบสามารถบันทึกข้อมูลคำสั่งซื้อ ชื่อลูกค้า และรายการสินค้าได้อย่างถูกต้องและแม่นยำ
3. **เพื่อเพิ่มประสิทธิภาพในการบริหารจัดการร้านค้า:** สร้างระบบหลังบ้าน (Admin Panel) ให้แอดมินสามารถเพิ่ม-ลดสินค้า ปรับปรุงราคาสินค้า และดูประวัติคำสั่งซื้อทั้งหมดได้ในที่เดียว
4. **เพื่อแจ้งเตือนข้อมูลอย่างรวดเร็ว (Real-time Notification):** เมื่อลูกค้าทำรายการสั่งซื้อสำเร็จ ระบบจะส่งข้อความแจ้งเตือนคำสั่งซื้อไปยังกลุ่ม LINE ของพนักงาน (Admin Group) ทันที เพื่อให้พนักงานเตรียมสินค้าได้อย่างรวดเร็ว
5. **เพื่อรองรับการขยายตัวของธุรกิจในอนาคต:** วางโครงสร้างระบบฐานข้อมูลและการทำงานแบบออนไลน์ (Cloud) ที่สามารถทนทานต่อปริมาณข้อมูลออเดอร์ที่เพิ่มขึ้นในอนาคต

---

## 3. ขอบเขตของระบบ (Scope)

ขอบเขตการพัฒนาระบบถูกแบ่งออกเป็น 2 ส่วนหลัก ได้แก่ ส่วนของลูกค้า (Customer / Frontend) ที่เชื่อมกับแพลตฟอร์ม LINE และส่วนของผู้ดูแลระบบ (Admin / Backend) ดังรายละเอียดต่อไปนี้:

### 3.1 ขอบเขตสำหรับผู้ใช้งาน (Customer Scope)
ผู้ใช้งานในที่นี้คือลูกค้าที่แอดไลน์ของทางร้านเข้ามาระบบจะรองรับการทำงานดังนี้:
- **ระบบระบุตัวตนผ่าน LINE (LIFF):** ลูกค้าสามารถกดลิงก์จาก Rich Menu หรือข้อความแล้วเข้าระบบได้ทันที โดยระบบจะดึงข้อมูลโปรไฟล์ (ชื่อผู้ใช้ใน LINE, User ID) เพื่อใช้เป็นฐานข้อมูลลูกค้า
- **การเรียกดูสินค้า (Product Catalog):** ลูกค้าสามารถค้นหาสินค้า ดูรายละเอียด ราคา และภาพประกอบของสินค้าที่วางขายได้
- **ระบบตะกร้าสินค้า (Shopping Cart):** ลูกค้าสามารถเพิ่ม ปรับลดจำนวน และลบสินค้าออกจากตะกร้าได้ พร้อมทั้งระบบคำนวณราคาสุทธิ
- **การยืนยันคำสั่งซื้อ (Checkout Process):** ลูกค้าสามารถยืนยันคำสั่งซื้อเพื่อส่งข้อมูลกลับไปยังระบบหลังบ้าน
- **การตรวจสอบสถานะออเดอร์:** ลูกค้าสามารถดูประวัติคำสั่งซื้อของตนเองที่ผ่านมาได้

### 3.2 ขอบเขตสำหรับผู้ดูแลระบบ (Admin Scope)
ผู้ดูแลระบบคือเจ้าของร้านหรือพนักงานที่มีหน้าที่จัดการข้อมูลต่างๆ ดังนี้:
- **การเข้าสู่ระบบแบบปลอดภัย (Admin Authentication):** แอดมินต้องทำการเข้าสู่ระบบ (Login) ด้วย Username/Password เพื่อเข้าถึงข้อมูล หรือเข้าสู่ระบบด้วย LINE Login
- **การจัดการสินค้า (Product Management):** แอดมินสามารถบันทึกข้อมูลสินค้าใหม่ (รหัสบาร์โค้ด, ชื่อ, ราคา, หมวดหมู่), อัปโหลดรูปภาพ, แก้ไข, และลบรายการสินค้าออกจากระบบ
- **การจัดการคำสั่งซื้อ (Order Management):**
  - แอดมินสามารถเรียกดูหน้า Dashboard สรุปรายการคำสั่งซื้อเรียงตามวัน-เวลาที่ลูกค้าสั่ง
  - สามารถเปลี่ยนสถานะออเดอร์ (เช่น "รอตรวจสอบ", "เตรียมจัดส่ง", "สำเร็จ", "ยกเลิก")
- **การแจ้งเตือนคำสั่งซื้อใหม่ (LINE Notification):** ระบบสามารถจำลองบอทเพื่อส่งสรุปออเดอร์เข้ากลุ่มแชทของพนักงานแบบอัตโนมัติ (Integration with LINE Messaging API)

### 3.3 ขอบเขตของระบบที่ไม่ได้ครอบคลุม (Out of Scope)
ระบบในเวอร์ชันแรก (Phase 1) **ยังไม่ครอบคลุม** ถึงการทำงานดังต่อไปนี้:
- ระบบระบบการชำระเงินอัตโนมัติหรือตัดบัตรเครดิตผ่าน Payment Gateway (ลูกค้ายืนยันออเดอร์ แล้วใช้วิธีแนบสลิปโอนเงินหรือชำระปลายทางแยกต่างหาก)
- ระบบสะสมแต้ม (Point/Loyalty Program)
- ระบบการคำนวณค่าจัดส่งอิงตามน้ำหนักและระยะทาง (ค่าจัดส่งแบบ Dynamic)
- การจัดการสต็อกและการทำงานแบบดึงข้อมูลข้ามคลังสินค้า (Multi-warehouse Inventory)
